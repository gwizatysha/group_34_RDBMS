# Group 34 - ALU School Database Management System

This repository contains Group 34's submission for the *Play with SQL Basics (RDBMS) - Peer Group Engagement Activity*.

## Overview
This project demonstrates the design and implementation of a relational database for a school management system using MySQL. It applies core database concepts including table design, normalization, primary and foreign key relationships, and SQL data manipulation.

## Database Structure
The database (`alu_project`) contains the following tables:
- **Students** - Stores student information.
- **Classroom** - Stores classroom details.
- **Faculty** - Stores faculty information.
- **Courses** - Stores course information, linked to Faculty and Classroom.

## Team Roles
- Member A: Students table
- Member B: Classroom table
- Member C: Faculty table
- Member D: Courses table
- Member E: Extra_Curricular_Activities table + junction tables (Student_Courses, Student_Activities)

## Repository Contents
- `README.md` - Project documentation
- `schema.sql` - Complete SQL script containing:
  - Database and table creation (DDL)
  - Primary and foreign key constraints
  - Sample data insertion
  - UPDATE, DELETE, and SELECT statements per member

## Technologies Used
MySQL, SQL, Git, GitHub

## Learning Outcomes
Through this project, we practiced relational database design, SQL programming, table relationships using foreign keys, and collaborative development using Git version control.
